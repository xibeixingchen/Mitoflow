# Quick Task 260505-gq6: Harden Backend Security

**Mode:** quick
**Date:** 2026-05-05

## Task Description

Fix P1 backend security and modernization issues identified in REVIEW.md:

1. CORS `allow_origins=["*"]` → configurable whitelist
2. PBKDF2 iterations 100k → 600k (OWASP 2023), with backward compatibility
3. FastAPI `@app.on_event("startup")` → `lifespan` context manager

---

## must_haves

- CORS reads allowed origins from `MITOFLOW_CORS_ORIGINS` env var (comma-separated)
- Default CORS behavior is same-origin only (no `*`)
- PBKDF2 uses 600,000 iterations for new passwords
- Old passwords with 100,000 iterations still verify correctly
- FastAPI startup uses `lifespan` context manager; `_ai_service` init moved there

---

## Tasks

### Task 1: Configurable CORS whitelist

**Files:** `deploy/web/backend/main.py`
**Action:**
- Replace hardcoded `allow_origins=["*"]` with env-driven list
- Read `MITOFLOW_CORS_ORIGINS` (comma-separated), default to `["http://localhost:5173", "http://localhost:8000"]`
- Split and strip whitespace

**Verify:**
- Without env var, only localhost origins allowed
- With `MITOFLOW_CORS_ORIGINS=http://a.com,http://b.com`, both allowed

**Done when:** CORS is no longer `*` in production

### Task 2: Upgrade PBKDF2 iterations

**Files:** `src/mitoflow/ai/auth.py`
**Action:**
- `_hash_password`: use 600,000 iterations
- `_verify_password`: auto-detect iteration count from stored hash format
- Hash format: `salt_hex:iterations:dk_hex` (new) or `salt_hex:dk_hex` (legacy)

**Verify:**
- New registration creates hash with `:` count = 3 (salt:iterations:dk)
- Login with old 2-part hash still succeeds
- Login with new 3-part hash succeeds

**Done when:** new passwords use 600k iterations, old passwords still work

### Task 3: Migrate FastAPI startup to lifespan

**Files:** `deploy/web/backend/main.py`
**Action:**
- Replace `@app.on_event("startup")` with `asynccontextmanager` lifespan
- Move `_ai_service` and `_ai_store` initialization into lifespan
- Move `_load_sessions()` call into lifespan

**Verify:**
- Server starts without error
- AI chat endpoints work after startup
- Sessions list is populated on first request

**Done when:** no `@app.on_event` decorators remain in main.py
