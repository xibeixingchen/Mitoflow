**C3 -- Session claiming race condition (ai.py:351-370):**
Replace the TOCTOU pattern in `_ensure_session_for_chat()`. Currently it calls `get_session_owner()` then `claim_session()` in two separate steps. Change to:
```python
def _ensure_session_for_chat(session_id: str, user: dict) -> None:
    if state.ai_store is None:
        return
    if not state.ai_store.session_exists(session_id):
        state.ai_store.create_session(session_id, user_id=user["id"])
        return
    owner = state.ai_store.get_session_owner(session_id)
    if owner is None:
        # Atomic claim -- returns False if already claimed by someone else
        if not state.ai_store.claim_session(session_id, user["id"]):
            raise HTTPException(status_code=403, detail="Session belongs to another user")
        return
    if owner != user["id"]:
        raise HTTPException(status_code=403, detail="Session belongs to another user")
```
The `claim_session()` method in `sessions_sqlite.py` already uses `UPDATE ... WHERE user_id IS NULL` which is atomic. The fix here is just to check the return value instead of blindly trusting the pre-check.

```python
        self.client = anthropic.Anthropic(**kwargs)
```

Remove the entire `saved_env` / `os.environ.pop` / `try/finally/restore` block. The anthropic SDK prioritizes explicit `api_key` over env vars when both are present, so the explicit api_key parameter is sufficient.

**M2 -- Blob URL memory leak (api/files.ts:67):**
The `previewFile` function creates a blob URL for binary files but never revokes it. Fix by returning a wrapper that tracks and revokes on next call:
```typescript
let _currentPreviewUrl: string | null = null

export function previewFile(name: string, sessionId: string): Promise<string> {
  const isText = /\.(fasta|fa|fas|fna|gb|gbk|gff|gff3|gtf|txt|csv|tsv|vcf|nwk|treefile|sam|json|xml)$/i.test(name)
  return apiClient
    .get(`/files/download/${name}`, {
      params: { session_id: sessionId },
      responseType: isText ? 'text' : 'blob',
    })
    .then((res) => {
      // Revoke previous preview blob URL to prevent memory leak
      if (_currentPreviewUrl) {
        window.URL.revokeObjectURL(_currentPreviewUrl)
        _currentPreviewUrl = null
      }
      if (res.data instanceof Blob) {
        const url = window.URL.createObjectURL(res.data)
        _currentPreviewUrl = url
        return url
      }
      return res.data as string
    })
}
```
Also update `file.ts` store's `previewFileAction` to revoke the old URL when preview changes. Add cleanup in the `previewFileAction`:
```typescript
async function previewFileAction(name: string, sessionId: string) {
```
